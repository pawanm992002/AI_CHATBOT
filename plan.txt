# Plan: JWT Cookie-Based Authentication

## Overview
Move JWT from localStorage + Authorization header to HttpOnly, Secure, SameSite cookies.
Widget API key auth stays as Bearer header (unchanged).

## Backend Changes

### 1. backend/core/auth.py
- Add set_auth_cookie(response, token, cookie_name="access_token")
  - HttpOnly=true, secure=settings.COOKIE_SECURE, samesite="lax", max_age=604800, path="/"
- Add clear_auth_cookie(response, cookie_name="access_token")
  - Sets max_age=0 to clear
- Rewrite get_current_tenant to read JWT from request.cookies.get("access_token")
  - Add Request parameter, remove HTTPBearer dependency
- Move get_current_admin here, rewrite to read from request.cookies.get("admin_token")

### 2. backend/routers/tenants.py
- POST /register — add response: Response param, call set_auth_cookie(response, token)
- POST /login — add response: Response param, call set_auth_cookie(response, token)
- Add POST /logout — calls clear_auth_cookie(response), returns {"message": "logged out"}

### 3. backend/routers/admin.py
- POST /admin/login — add response: Response, call set_auth_cookie(response, token, "admin_token")
- Add POST /admin/logout — calls clear_auth_cookie(response, "admin_token")
- Move get_current_admin to auth.py, rewrite to read from cookie via Request

### 4. backend/main.py
- No structural changes. CORS middleware already reflects origin + credentials.

## Frontend Changes

### 5. dashboard/src/utils/axios.ts
- Remove request interceptors that inject Authorization header from localStorage
- Add withCredentials: true to privateAxios and adminAxios
- Keep response interceptors for 401/403 redirect logic

### 6. dashboard/src/api.ts
- clearSession() — remove localStorage.removeItem('token')
- redirectToLogin() — call POST /logout before redirect

### 7. dashboard/src/App.tsx
- PrivateRoute — replace localStorage check with GET /tenants/me on mount
  - 401 -> redirect to login, 200 -> render children, loading -> spinner
- AdminRoute — same pattern with GET /admin/tenants

### 8. dashboard/src/pages/Login.tsx
- Remove localStorage.setItem('token', ...) after login

### 9. dashboard/src/pages/AdminLogin.tsx
- Remove localStorage.setItem('adminToken', ...) after login

## Cookie Specifications

| Property    | Tenant Cookie     | Admin Cookie      |
|-------------|-------------------|-------------------|
| Name        | access_token      | admin_token       |
| HttpOnly    | true              | true              |
| Secure      | from config       | from config       |
| SameSite    | lax               | lax               |
| MaxAge      | 604800 (7 days)   | 604800 (7 days)   |
| Path        | /                 | /                 |

## Unchanged
- Widget Bearer auth (verify_api_key) — no change
- chat_session_id cookie — no change (visitor session tracking)
- JWT creation logic (create_access_token) — no change
- bcrypt password hashing — no change
